💾 ARMAZENAMENTO
───────────────────────────────────────────────────────────────────────────────

Cada semana o agente coleta ~150MB de dados que são processados em:

  • ~1-5 MB em embeddings (ChromaDB) ← Memória persistente
  • ~500KB em cache (TTL 6 horas)
  • ~20MB em logs (com rotação)

Resultado: Banco de dados eficiente que cabe em qualquer servidor!

💡 DICAS PROFISSIONAIS
───────────────────────────────────────────────────────────────────────────────

🔑 CONFIGURAR FRED (Gratuito - Recomendado)
   1. Vá para: https://fred.stlouisfed.org/docs/api
   2. Registre-se (gratuito)
   3. Obtenha sua chave API
   4. Coloque em .env: FRED_API_KEY=sua_chave

📈 MONITORAR PERFORMANCE
   - Verifique logs/macroeconomist.log
   - Veja task_stats para taxa de sucesso
   - Use memory.get_collection_stats() para monitorar memória

🚀 DEPLOY EM PRODUÇÃO
   - Use systemd (Linux) ou Task Scheduler (Windows)
   - Configure logs com rotação automática
   - Monitore com prometheus ou similar
   - Backup regular do ./data/chroma_db/
