════════════════════════════════════════════════════════════════════════════
PASSO 1: PEGAR CONNECTION STRING DO SUPABASE (⏱️ 2 minutos)
════════════════════════════════════════════════════════════════════════════

1. Acesse: https://supabase.com/dashboard
   (Faça login com sua conta)

2. Escolha seu projeto (se tiver mais de um)

3. No menu ESQUERDO, procure: "Settings" ou "Configurações"

4. Clique em: "Database" ou "Banco de Dados"

5. Procure a seção: "Connection string" ou "URI de Conexão"

6. Você vai ver algo assim:

   postgresql://postgres.<project-ref>:<db-password>@db.<project-ref>.supabase.co:5432/postgres

   Copie TODA essa string (Ctrl+C)


════════════════════════════════════════════════════════════════════════════
PASSO 2: CONFIGURAR .env (⏱️ 2 minutos)
════════════════════════════════════════════════════════════════════════════

1. Nesta pasta, procure arquivo: .env.example

2. Se não existir .env ainda:
   - Copie .env.example e renomeie para .env

3. Abra o arquivo .env no Notepad ou VS Code

4. Procure esta linha:

   # SQLALCHEMY_DATABASE_URL=postgresql://...

   (Pode estar comentada com # no começo)

5. Substitua por (cola a string do Supabase):

   SQLALCHEMY_DATABASE_URL=postgresql://postgres.<project-ref>:<db-password>@db.<project-ref>.supabase.co:5432/postgres

   (Sem o # na frente!)

6. Se não tiver a linha, adicione logo embaixo de todas as outras configs:

   SQLALCHEMY_DATABASE_URL=<cole_aqui_sua_connection_string>

7. Salve (Ctrl+S)


════════════════════════════════════════════════════════════════════════════
PASSO 3: INSTALAR DEPENDÊNCIAS (⏱️ 2 minutos)
════════════════════════════════════════════════════════════════════════════

Abra PowerShell nesta pasta e execute:

    pip install -r requirements.txt

Espere terminar (2 minutos).


════════════════════════════════════════════════════════════════════════════
PASSO 4: CRIAR TABELAS NO SUPABASE (⏱️ 2 minutos)
════════════════════════════════════════════════════════════════════════════

Abra PowerShell nesta pasta e execute:

    python setup_postgresql.py

Este script vai:
  ✓ Conectar no Supabase
  ✓ Criar tabelas automaticamente
  ✓ Criar índices (para speed)
  ✓ Mostrar se deu certo

Se vir "✅ SETUP COMPLETO COM SUCESSO!" = funcionou! 🎉


════════════════════════════════════════════════════════════════════════════
PASSO 5: TESTAR SE FUNCIONA (⏱️ 5 minutos)
════════════════════════════════════════════════════════════════════════════

Abra PowerShell nesta pasta e execute:

    python professional_examples.py

Isto vai testar 7 coisas:
  1. Coleta de dados (vai pedir API key da FRED)
  2. Teste de estacionaridade
  3. Causalidade Granger
  4. Correlação
  5. Volatilidade
  6. Previsão
  7. Elasticidade

Se vir "✅ TODOS OS EXEMPLOS EXECUTADOS COM SUCESSO!" = tudo pronto! 🎉

(Pode ter warnings, é normal. Erros vermelhos = problema)


════════════════════════════════════════════════════════════════════════════
PROBLEMA: FRED API KEY
════════════════════════════════════════════════════════════════════════════

Você vai precisar de uma chave FRED para coleta de dados (é GRÁTIS):

1. Acesse: https://fred.stlouisfed.org/docs/api/api-key.html

2. Clique: "Request API Key"

3. Preencha:
   • Email: seu_email@gmail.com
   • First name: seu nome
   • Last name: seu sobrenome
   • Affiliation: "Personal" está OK

4. Vão mandar um email com seu API KEY

5. Copie a chave

6. Abra .env e procure:

   FRED_API_KEY=seu_fred_api_key_aqui

7. Mude para:

   FRED_API_KEY=sua_chave_aqui

8. Salve (Ctrl+S)

(Demora uns 2 minutos para ativar)

════════════════════════════════════════════════════════════════════════════
SE DER ERRO
════════════════════════════════════════════════════════════════════════════

❌ "Connection refused to Supabase"
✅ Verifique:
   • Connection string está correta no .env
   • Não cortou nada da string
   • Internet está conectada

❌ "SSL: CERTIFICATE_VERIFY_FAILED"
✅ Algumas redes bloqueiam. Tente adicionar ao .env:
   SQLALCHEMY_ECHO=False

❌ "ModuleNotFoundError: No module named 'statsmodels'"
✅ Execute novamente: pip install -r requirements.txt

❌ "Table doesn't exist"
✅ Execute: python setup_postgresql.py

❓ Outro erro?
✅ Copie a mensagem exata e me mostra!
